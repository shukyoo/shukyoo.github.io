SSL Connect Error Solution

2017-05-15

curl -v ‘https://mpos.quanminfu.com:6004/merFrontMgr/orderQuerySerlet‘

  • About to connect() to mpos.quanminfu.com port 6004 (#0)
  • Trying 180.169.95.130… connected
  • Connected to mpos.quanminfu.com (180.169.95.130) port 6004 (#0)
  • Initializing NSS with certpath: sql:/etc/pki/nssdb
  • CAfile: /etc/pki/tls/certs/ca-bundle.crt
    CApath: none
  • NSS error -12173
  • Closing connection #0
  • SSL connect error
    curl: (35) SSL connect error

引起：
升级了nss（Network Security Service），从3.15 -> 3.19，导致全民付ssl无法连接
NSS error -12173 : SSL handshake failed

直接原因：
全民付ssl版本过低

解决方法：
curl_setopt($curl, CURLOPT_SSL_CIPHER_LIST, ‘rsa_aes_128_sha’);

2016-05-26
PHP

PDO的FETCH_*方面的常量对读取数据方面非常实用，以下常量通过参考PHP手册以及参考网上资料自己测试和总结而来。

  • FETCH_NUM：获取每一行以字段位置作为索引的数组，从0开始；
  • FETCH_ASSOC：获取每一行以字段名作为索引的数组；
  • FETCH_NAMED：跟FETCH_ASSOC类似，只是当遇到结果集中存在字段重名时，FETCH_NAMED针对重名字段返回的是数组形式，而FETCH_ASSOC针对重名字段只返回一个值；
