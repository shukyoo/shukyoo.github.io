错误和异常

2017-02-09
Thinking

我们在程序里经常会碰到错误和异常，那么这里就会引申出几个问题：

  1. 错误和异常的区别是什么；
  2. 为什么要有错误和异常的区分；
  3. 什么时候用错误，什么时候用异常，是否可以全部用异常然后通过try cache实现错误表示；

我们举几个例子再引申思考一下：

  1. 用户登录输错密码，这个是错误还是异常？
  2. 用户充值10元，结果给用户充了20元进去，这个是错误还是异常？
  3. 用户要上传文件，结果因为文件格式不正确上传失败了，这个是错误还是异常？又如果因为网络原因上传失败了，这个是错误还是异常？
  4. 开发在实例一个User类的时候，要传int类型的id值进行构造，结果传了一个字符串，这个是错误还是异常？
  5. 程序要调用第三方接口，结果因各种原因调用失败，这个是错误还是异常？
  6. 数据保存失败，这个是错误还是异常？

错误和异常在某些方面是一样的，都表示了在正常业务逻辑之外的可能存在的旁支逻辑；
错误分为两种维度的错误，一种维度是产品层面的错误，属于正常的业务范畴，比如用户密码错误、上传文件类型不正确；另一种维度是程序层面的错误，这种可能更接近于异常，所以我们要比较的更多是程序层面的错误和异常；

首先我们要确定一个概念就是“错误是值，异常是真正的错误”，错误是可预期可接受的，异常是不可预期不可接受的；
对于可预期可接受的逻辑，我们往往能对它作后续的准备和处理，但是对于不可预期不可接受的逻辑，我们往往没有办法，只能throw exception让程序崩溃让它速错；

在构造方法里出现的错误，比如传入的参数，对于可接受可容忍的错误，我们会让参数以比如默认值的形式赋值并继续，但是对于不可容忍的错误，我们只能throw exception；

在链式模式下出现的错误，因为链式模式的返回是this，所以一般情况的错误都会throw exception，要确保返回this，否则链式就会断；

错误表现为宽容的处理，通过人为干预，让程序继续；异常表现为阻止和不处理，如果继续意味着真正的业务错误，所以让程序崩溃；

对于底层类库的throw exception，一般是为了“防止使用不当”和“程序异常”而设置的“阻止”，并通过速错让错误暴露，比如

比如

if (!$this->id) {
return false;
}

function dosome() {
if (empty($data)) {
return [‘status’ => false, ‘msg’ => ‘something error’];
}
return [‘status’ => true, ‘data’ => $data];
}

function dosome(&$errmsg = null) {
if (empty($data)) {
$errmsg = ‘something error’;
return false;
}
return true;
}

class Test
{
protected $_errmsg;

public function dosome() {
    if (empty($data)) {
        $this->_errmsg = 'something error';
        return false;
    }
    return true;
}

public function getErrmsg() {
    return $this->_errmsg;
}

}
```
这里不提倡用方式1，像golang通过多返回值的形式，把error作为返回值的其中一个是可以的，但是只有一个返回值而返回带状态的数组这样的设计是糟糕的，除非返回值明确是为了返回错误信息；

以上是一般原则，在实际开发中错误和异常会混用，在混用时候要以一般原则作为参考，结合实际情况而定；

MySQL复合索引使用需知

2016-05-24
Database

复合索引 是指用多个字段创建一条索引；

使用复合索引的时候，需要了解以下几点：

  1. MySQL每次查询只能使用一个索引，因此如果多个条件同时查询的频率较高的情况下可以考虑创建复合索引；
  2. 只有当复合索引的第一个字段出现在条件查询中，复合索引才能生效，称为复合索引的前缀性；
  3. 创建了(A, B, C)这样一个复合索引，相当于覆盖了(A, B, C), (A, B), (A)三个索引，即第2点，A必须出现在条件查询中，另外把查询频繁从高到低，对应复合索引字段从左到右排列，称为最佳左前缀特性；
  4. 不要创建过多的索引，一方面参考第1点（每次查询只能使用一个索引），另一方面过多的索引会影响表的更新效率；
  5. 复合索引和其它类型索引各有优势，因此要按实际情况来选择和创建索引；
