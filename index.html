做一个高效的程序员

2016-05-28
Practice

我们要懒出水平，更多的事情要让机器去做。

《卓有成效的程序员》这本书可以看一下。

基于我的实践和感悟，以下是我的认识和总结：

首先我们需要有好的硬件；

  • 高配电脑和好的系统，首推Mac笔记本或MiniMac，其次是Linux，最后才是Windows；
  • 最好有两台显示器；
  • 机械键盘，我们可能是敲键盘最多的行业；

多用键盘，少用鼠标（任何事都是开头难）

  • 并不是不用鼠标，而是指多使用快捷键（系统、IDE、浏览器等等），写代码的时候可以手不离键盘，它能让你有飞一般的感觉；
  • 规范你的键盘手势（你的小拇指有没有用起来？）如果不规范请苦练并习惯，它能让你全盘掌控你的键盘；
  • 建议把Ctrl键和CapsLock键互换，当你经常用快捷键的时候你就懂了（如果你使用Emacs的话，这是必须要做的）

好的IDE能事半功倍

  • 各种语言都有不同的优秀的IDE，找到一款好的并适合自己的IDE很重要，比如PHP开发我推荐是PHPStorm+Sublime/VSCode；
  • 后端开发，至少需要熟悉vim或emacs的使用和常用快捷键（emacs比较强大，但是前期学习难度大）；

好的软件和工具
不同的开发使用的工具都会不一样，我推荐几个我了解的；

  • 本地搜索和调度：Mac - Alfred，Linux - Mutate，Windows - Listary；
  • SSH：SecureCRT+SecureFX；
  • git托管：github, bitbucket, coding, oschina；
  • 文档：Google Doc，石墨文档，git；
  • TodoList：Tower；
  • 沟通：HipChat，瀑布，BearyChat；

git不只用来作代码托管

  • 做备份；
  • 做博客Host；
  • 做Wiki文档；
  • 做QA；
  • 做Code Review；

把常用命令做成alias或shell脚本

  • 熟悉常用的shell命令；
  • 然后把常用命令做成alias或shell脚本，减少重复劳动；

多使用Google搜索

  • 一定的英语基础是程序员的必备技能，要会阅读和英文搜索；
  • 怎么上Google这个我不多说了；
  • github, stackoverflow怎能没听说过？快用起来吧；

DRY（Don’t Repeat Yourself）

  • 很多好的框架，好的开源代码为什么不拿来用，如果哪里不满意可以为开源代码提供自己的智慧；
  • 你确定你写的系统比上百上万团队都在使用的开源系统要更好？
  • 为什么要去做重复的劳动？这是一个快鱼吃慢鱼的时代；
  • 各种语言都有依赖管理工具，要好好用起来，比如PHP的composer；

自动化和容器

  • 比如使用Jenkins作持续集成和自动部署；
  • 做一些自动化的脚本和工具，处理实践中的重复劳动；
  • 比如Docker虚拟容器；
  • IT和运维自动化（专业知识请自行了解）

自我养成和与时俱进

  • 自我规范，代码规范，带一些强迫症；
  • 良好的时间观念和良好的文件管理；
  • 多思考与学习，保持与时俱进，保持创新和热情；
  • 开发所涉及的东西太多了，变化太快了，不要固执于自己已知的语言和知识，多学习和实践；

PDO::FETCH_*常量解释和示例

2016-05-26
PHP

PDO的FETCH_*方面的常量对读取数据方面非常实用，以下常量通过参考PHP手册以及参考网上资料自己测试和总结而来。

  • FETCH_NUM：获取每一行以字段位置作为索引的数组，从0开始；
  • FETCH_ASSOC：获取每一行以字段名作为索引的数组；
  • FETCH_NAMED：跟FETCH_ASSOC类似，只是当遇到结果集中存在字段重名时，FETCH_NAMED针对重名字段返回的是数组形式，而FETCH_ASSOC针对重名字段只返回一个值；

MySQL复合索引使用需知

2016-05-24
Database

复合索引 是指用多个字段创建一条索引；

使用复合索引的时候，需要了解以下几点：

  1. MySQL每次查询只能使用一个索引，因此如果多个条件同时查询的频率较高的情况下可以考虑创建复合索引；
  2. 只有当复合索引的第一个字段出现在条件查询中，复合索引才能生效，称为复合索引的前缀性；
  3. 创建了(A, B, C)这样一个复合索引，相当于覆盖了(A, B, C), (A, B), (A)三个索引，即第2点，A必须出现在条件查询中，另外把查询频繁从高到低，对应复合索引字段从左到右排列，称为最佳左前缀特性；
  4. 不要创建过多的索引，一方面参考第1点（每次查询只能使用一个索引），另一方面过多的索引会影响表的更新效率；
  5. 复合索引和其它类型索引各有优势，因此要按实际情况来选择和创建索引；

在你往上爬的时候，一定要保持梯子的整洁，否则你下来时可能会滑倒。

测验一个人的智力是否属于上乘，只看脑子里能否同时容纳两种相反的思想，而无碍于其处世行事。

刺猬理论
刺猬在天冷时彼此靠拢取暖，但保持一定距离，以免互相刺伤。

鲦鱼效应
鲦鱼因个体弱小而常常群居，并以强健者为自然首领。将一只稍强的鲦鱼脑后控制行为的部分割除后，此鱼便失去自制力，行动也发生紊乱，但其他鲦鱼却仍像从前一样盲目追随。
