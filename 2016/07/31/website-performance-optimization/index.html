网站性能优化

2016-07-31
Architecture

一个网站的性能优化可以从前到后来分析和优化。
优化分两方面，一方面是开发层面的优化，另一方面是架构层面的优化，这里主要是指架构层面的优化，也即对现有网站的性能优化策略。
优化的实现细节很多，这里只列出关键点，至于实现则都是文章了。
从前到后我们分四个方面：
前端性能优化，后端性能优化，数据库性能优化和服务器架构

一，前端性能优化

  • 减少http请求
  • 压缩静态资源
  • 增加静态资源域名
  • 异步加载
  • Lazy Load（JS和图片）
  • 本地缓存
  • 压缩图片
  • gzip传输
  • 压缩html
  • css sprite
  • JS脚本放到页脚
  • 避免html图片缩放（大图当小图用）

二，后端性能优化（这里更多是实现上的注意点）

  • 避免联表查询
  • 避免循环查数据库
  • 页面缓存+内容缓存
  • 避免多重循环
  • 异步处理
  • 单例设计模式
  • 避免过度封装
  • 避免重复取数据

PHP层面：

  • 熟悉和使用php内置函数
  • 升级php版本
  • 使用xhprof查找瓶颈
  • Opcode cache

三，数据库性能优化

  • 合理使用索引
  • 解决慢查询
  • 分表分库(水平拆分)
  • 避免大表（垂直拆分）
  • SQL语句优化
  • 避免全表扫描

四，服务器架构

  • 负载均衡
  • 主从数据库
  • 分布式缓存
  • 消息队列
  • CDN
  • 大请求服务分离
